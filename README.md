# ha_sonnenheater

A Home Assistant integration that scrapes data from your sonnenHeater out of the official sonnen Customer Portal. 

---

## Notes

- Credentials are never stored by this tool. Use environment variables or pass them at runtime.

---

## Recommended usage
- Run a docker container with the server that constantly scrapes data
- Install the Home Assistant custom component

---

## REST API Server

`server.py` wraps the scraper in a FastAPI application. It scrapes on a configurable interval and caches the result so that Home Assistant (or any other client) can poll it without launching a new browser session every time.

The image is automatically built and published to the GitHub Container Registry on every push to `main` and on version tags:

```bash
docker pull ghcr.io/f0n51/ha_sonnenheater:latest
```

### Start the server (Docker Compose — recommended)

```bash
# 1. Copy the example env file and fill in your credentials
cp .env.example .env

# 2. Edit .env with your SONNEN_USERNAME and SONNEN_PASSWORD

# 3. Start (pulls the pre-built image from GHCR)
docker compose up -d
```

The API is then available at `http://localhost:8099/data` (or replace `localhost` with your host IP for Home Assistant access).

To view logs:

```bash
docker compose logs -f
```

To stop:

```bash
docker compose down
```

### API

| Method | Path | Description |
|---|---|---|
| `GET` | `/data` | Returns the latest scraped data as JSON |
| `GET` | `/health` | Health check — `200 ok` when last scrape succeeded, `503 degraded` on error |
| `GET` | `/docs` | Interactive Swagger UI (auto-generated by FastAPI) |
| `GET` | `/` | Redirects to `/docs` |

## Home Assistant Integration

The `custom_components/sonnenheater/` folder is a HA custom integration that reads the scraped sonnenHeater data from the server above.

### Installation via HACS (recommended)

1. Make sure [HACS](https://hacs.xyz) is installed in your Home Assistant instance.
2. Go to **HACS → Integrations → ⋮ (top-right menu) → Custom repositories**.
3. Paste `https://github.com/f0n51/ha_sonnenheater` and select category **Integration**, then click **Add**.
4. Find **Sonnen Heater** in the HACS integration list and click **Download**.
5. Restart Home Assistant.
6. Go to **Settings → Devices & Services → Add Integration** and search for **Sonnen Heater**.
7. Enter the server URL (e.g. `http://192.168.1.100:8099/data`) and the poll interval.

To change the server URL or poll interval later, click Configure on the Sonnen Heater integration card.

<details>
<summary>Manual installation</summary>


1. Copy `custom_components/sonnenheater/` into your HA config directory:
   ```
   /config/custom_components/sonnenheater/
   ```
2. Restart Home Assistant.
3. Go to **Settings → Devices & Services → Add Integration** and search for **Sonnen Heater**.
4. Enter the server URL (e.g. `http://192.168.1.100:8099/data`) and the poll interval.
</details>

### Entities

All entities appear under a single **Sonnen Heater** device.

| Entity | Unit | Device class |
|---|---|---|
| Heater Power | W | `power` |
| Heater Water Temperature | °C | `temperature` |
| Heater Max Temperature | °C | `temperature` |
| Heater State | — | text (`active` / `inactive`) |

## Python scripts
Two modes are available:

- **`sonnenbatterie_scraper.py`** — one-shot CLI tool, prints or saves JSON and exits.
- **`server.py`** — long-running REST API server that polls on a schedule and serves the latest data at `GET /data`. Used as the back-end for the Home Assistant custom integration.

### Output

Returns a JSON object with two sections:

```json
{
  "battery_info": {
    "serial_number": "12345",
    "capacity": "10.0 kWh",
    "model": "sonnenbatterie 10",
    "operating_mode": "Self-Consumption"
  },
  "sonnen_heater": {
    "power": "2000 W",
    "water_temperature": "55 °C",
    "max_temperature": "60 °C",
    "state": "active"
  },
  "timestamp": "2026-04-03T10:00:00.000000"
}
```

### Prerequisites

- Python 3.9+
- Google Chrome / Chromium (installed via Playwright, see below)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/f0n51/ha_sonnenheater.git
cd ha_sonnenheater

# 2. Create and activate a virtual environment (recommended)
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. Install the Playwright browser
playwright install chromium
```

### Usage

#### With command-line flags

```bash
python sonnenbatterie_scraper.py -u your@email.com -p YourPassword
```

#### With environment variables

Copy `.env.example` to `.env`, fill in your credentials, then:

```bash
# Windows (PowerShell)
$env:SONNEN_USERNAME="your@email.com"
$env:SONNEN_PASSWORD="YourPassword"
python sonnenbatterie_scraper.py

# macOS/Linux
export SONNEN_USERNAME="your@email.com"
export SONNEN_PASSWORD="YourPassword"
python sonnenbatterie_scraper.py
```

#### Save output to a file

```bash
python sonnenbatterie_scraper.py -u your@email.com -p YourPassword -o data.json
```

#### Show the browser window (useful for debugging login issues)

```bash
python sonnenbatterie_scraper.py -u your@email.com -p YourPassword --visible
```

#### All options

| Flag | Short | Description |
|------|-------|-------------|
| `--username EMAIL` | `-u` | Login email (or `SONNEN_USERNAME` env var) |
| `--password PASSWORD` | `-p` | Login password (or `SONNEN_PASSWORD` env var) |
| `--output FILE` | `-o` | Write JSON output to a file instead of stdout |
| `--visible` | | Run browser in non-headless mode |
| `--debug` | | Save `debug_page.html` and `debug_screenshot.png` after rendering |





### Environment variables

| Variable | Default | Description |
|---|---|---|
| `SONNEN_USERNAME` | — | Login e-mail (required) |
| `SONNEN_PASSWORD` | — | Login password (required) |
| `POLL_INTERVAL_SECONDS` | `300` | How often to re-scrape (seconds) |
| `SERVER_PORT` | `8099` | TCP port the server listens on |
| `SERVER_HOST` | `0.0.0.0` | Bind address |
| `LOG_SCRAPED_DATA` | `false` | Set to `true` to log the full scraped payload to stdout after each poll |
